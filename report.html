Project 1 Expectations Breakdown

  1. You will create a website with a focused theme and audience of your choosing. Your site might serve a student group, a relative who’s planning a wedding, a business, or an artist who needs a portfolio. The site must serve someone besides yourself. A large part of web design is thinking empathetically, and this requirement makes that explicit.
    • This website was designed to serve fans of the Miami Heat basketball team. It provides them with a schedule, roster, news, and an animation to enjoy.
  2. Your content must be original and not lifted from other websites. Third-party images are acceptable provided you adhere to their license.
    • Any resources I found I used free licensing of logos and other images.
  3. Your site must contain at least five separate pages, each having a meaningful purpose. One of the files should be named index.html, which Apache loads by default.
    • index.html - Acts as my home page, explains the site layout and implementation of roster tiles to go to nba.com profile page
    • roster.html - Displays a list of Miami Heat players with stats
    • news.html - Displays recent news about the Miami Heat org and players
    • schedule.html - Displays future schedule of the team
    • animation.html - Displays animation of trophies falling
  4. Your site must include some sort of header.
    • There is a header of blurry Miami background and the Miami Vice logo in front of it.
  5. Your site must include some sort of navigation links.
    • Every page has a navigation bar with links to all html files of the site
  6. Your site must support distinct layouts for large and small screens using media queries. For instance, multicolumn layouts on large screens should be reduced to single column layouts on small screens.
    • All pages of the site dynamically resize and offer scrolling options, also removing content as the screen size limits
  7. Your site must maintain a coherent style across its pages. Repeated style rules must be placed in a shared external stylesheet. Style rules not shared between pages must be placed in separate external stylesheets.
    • main-style.css serves the function of coherent style and the style rules repeated. All other extenal stylesheets are appropriately named for their respective html file
  8. Your site must include images, and their alt attributes must adequately describe the image content to users of screen readers, user with images disabled, and webcrawlers.
    • All images have descriptions in the alt tags that adequately describe image content
  9. Each page must declare its favicon.
    • Each page's favicon is a custom edited svg of the Miami Heat logo that I created
  10. Your site’s non-HTML resources must be organized. Images must be placed in a directory named images. CSS files must be placed in a directory named css. If you include any JavaScript code, these files must be placed in a directory named js.
    • All assets and folders of the projet are neatly organized in project1
  11. Your site must not be built using any libraries or frameworks. Do not use Bootstrap, jQuery, React, Angular, Vue, or any other third-party tools. You may, however, reimplement the ideas you see in these third-party tools.
    • No files use any libraries or frameworks
  12. Each page and stylesheet of your site must validate without any warnings or errors using the W3C’s Nu Validator (Links to an external site.)
    • All html and css files pass the validator
  13. The content should be neatly aligned. The axes of alignment should be clearly visible. Favor left- and right-alignment over centering—as centering does not produce a strong axis of alignment.
    • All content is neatly aligned across all screen
  14. The content should have adequate and consistent spacing. Text should never bleed against the edge of its containing box.
    • Text never bleeds to the edge of it's box and content is adequately and consistently spaced
  15. Your site must include at least one form. The form need not do anything at this point in your learning, but it must be present. Like all content, it must be neatly aligned.
    • The form is included as the login at the top of the page
  16. Your site must include links to external sites.
    • The site links to mulitple, such as: nba.com, ticket purchasing sites, youtube, various news sources, etc
  17. Your source code must be readable, using conventional and consistent formatting and meaningful names for classes and IDs. Favor kebab-case case identifiers over camelCase.
    • Source code is readable, elements are meaningfully named and intuitive
  18. Your site must include at least one animation, powered either by CSS or JavaScript.
    • The site includes a CSS powered animation at animation.html of trophies falling
  19. Your site must be stored in a Git repository that you have shared with your instructor. On GitHub, GitLab, and Bitbucket, share with the user hcientist.

  20. The names of your files must be entirely lowercase and contain only alphanumeric characters.
    • All files match these restrictions
